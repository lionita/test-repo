# test-repo

Multi-module Spring Boot reference implementation using Java 22, Spring Boot 4.0.0, PostgreSQL, REST, Spring Data JPA, Hexagonal-ish layering, outbox pattern, and JWT-based Spring Security checks.

## Modules
- `auction-module`: auction aggregate and auction lifecycle use cases.
- `bidding-module`: bidding use cases and bid-specific ports.
- `app-module`: Spring Boot app wiring REST adapters + JPA adapters + outbox publisher.

## JWT security checks
- API endpoints require a bearer JWT.
- `POST /api/auctions` and `POST /api/auctions/{id}/start` require `scope` claim containing `auction.write`.
- `POST /api/auctions/{id}/bids` requires `scope` claim containing `bid.write`.
- `POST /api/bidders/onboarding` requires `scope` claim containing `admin.write`.
- `PUT /api/bidders/{bidderId}`, `DELETE /api/bidders/{bidderId}`, `POST /api/bidders/{bidderId}/block`, and `POST /api/bidders/{bidderId}/unblock` require `scope` claim containing `admin.write`.
- JWT must include a non-empty `sub` claim.

## OpenAPI documentation
- Source spec file: `docs/api.yaml`
- Swagger UI: `http://localhost:8080/swagger-ui.html`
- OpenAPI JSON: `http://localhost:8080/v3/api-docs`
- Both OpenAPI endpoints are publicly accessible; business API endpoints still require JWT.

## Bidder onboarding
- Before placing bids, each bidder must be onboarded via `POST /api/bidders/onboarding`.
- Admins can update bidder details via `PUT /api/bidders/{bidderId}` and soft delete bidders via `DELETE /api/bidders/{bidderId}`.
- Admins can time-block bidders via `POST /api/bidders/{bidderId}/block` (default configurable duration: 2 months) and manually unblock via `POST /api/bidders/{bidderId}/unblock` (both return `204 No Content`).
- Onboarding captures bidder profile details: `firstName`, `lastName`, `email`, and `nationalId`, along with `purchasingAuthorizationLimit`.
- `bidderId` is autogenerated by the system during onboarding and returned in the onboarding response together with bidder profile details.
- The JWT `sub` must match `bidderId` for bid placement requests.
- Bidder onboarding is performed by admin users (separate authority from bidding).
